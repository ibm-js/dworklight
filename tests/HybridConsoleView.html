Worklight Console Intercept Tests

This tests out the "issw/mobile/worklight/hybrid-console" module's ability to intercept console.xxx() and WL.Logger.xxx() calls.

The XCode (IOS) and LogCat (Android) logging environments only support a single string console logging argument. Passing extra arguments are ignored, and passing in an object produces "null". Not very helpful.

In hybrid applications, console interceptors are put in place by the worklight or cordova modules. These will accept all normal console arguments and attempt to merge and format objects as JSON strings. If running under Worklight, the Wl.Logger functions are also intercepted.

All output of these tests are shown in the appropriate logging environment. Look for Errors being thrown or any output that does not look proper.

The following console commands are run on each view of this test, being called with a; string, integer, boolean, null, object, and various combinations:

    console.debug();
    console.log();
    console.info();
    console.warn();
    console.error();
If a worklight-based hybrid application, these are also tested
    WL.Logger.debug();
    WL.Logger.error();
